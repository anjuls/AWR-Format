AWR Formatter

Primary Features

  1. Convert hard to read sizes such as "reads" and "gets" to KB / MB / GB / TB.
  2. Convert hard to read times such as thousands of seconds to hours / minutes / seconds.
  3. Link common wait events to their definitions. A local copy of the definitions is included so internet access is not necessary.
  4. Large HTML tables of data are sortable and searchable.
  5. Top SQL by Elapsed, Reads, Gets, and Executions are combined into one table.
  6. A SQL Popup that is called when you click on any SQL in the report. This shows the SQL with several formatting options, including the option for syntax highlighting to make it easier to read. New tabs were added in version 1.6:
    • Query Plan: Shows the statement to execute to query the AWR repository and return the plan for this SQL statement.
    • Color SQL: This shows the statement to execute to "color" a SQL statement. Once a statement is colored, it will always show up in the AWR report, regardless of whether it's in the top n SQL or not.
    • AWR SQL Report: Shows the statement to execute to generate an AWR report for a single query.
  7. A tabbed interface is added to the top to surface key information. The tabs include:
    • Overview: A concise list of key attributes and values
    • Top SQL Combined
    • Top Events: Data from the Top Events sections, such as Top 5 Timed Foreground Events, are shown together in one tab. Metrics below a certain threshold, such as less than 2% of DB Time, are filtered.
    • Memory
    • Observations: Certain metrics and values are worth additional investigation. These could include the use of undocumented parameters or an AWR Report that spans a period of time greater than 60 minutes. The observations are listed here and are also emphasized on the report with a red icon.
    • IO Graphs
  8. Each HTML table offers "CSV Export"
